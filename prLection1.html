УП.05: Сбор исходных данных для разработки ИС

Для ознакомления с предыдущей лекцией, перейдите по ссылке.

Кратко: Важность сбора данных для разработки проектной документации, этапы и методы, ошибки и рекомендации.

Основные блоки исходных данных

  • Организационно-экономические: устав, бизнес-процессы, проблемы, цели, бюджет, сроки.
  • Технические: инфраструктура, программное обеспечение, требования к интеграции и характеристикам.
  • Функциональные: сценарии использования, функции системы, бизнес-правила.
  • Пользовательские требования: интерфейс, эргономика, стиль.
  • Нормативно-правовые: законодательство, стандарты, требования к документам.

Методы сбора данных

  • Интервью, анкеты, наблюдение.
  • Анализ документов, семинары, прототипирование, мозговой штурм.

Пошаговый план сбора данных

  • Подготовка: цели, стейкхолдеры, план, шаблоны.
  • Сбор: интервью, документы, фиксация данных.
  • Анализ: обработка, схемы, требования.
  • Отчет: оформление, согласование, утверждение.

Ошибки и рекомендации

  • Игнорирование мнения пользователей — учитывать всех.
  • Разрозненные требования — анализировать и уточнять.
  • Пропуск согласования — фиксировать договоренности.
  • Один метод — использовать комплексный подход.
